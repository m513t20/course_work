# API взаимодействия со стендом
Для взаимодействия с базовым модулем компьютерного зрения на стенде развернут web-сервер со своим API.

**ВНИМАНИЕ** данное API используется при взаимодействии со стендом. Стучатся вне этого времени смысла нет

## Калибровка стенда
```
http://{stand_api}/calibrate
```

Где 
|аргумент|значение|
|--------|--------|
|`stand_api`|ip адрес стенда в локальной сети|
Запускает калибровку параметров стенда по аруко маркерам. До запуска нужно полностью заполнит левый столбец и нижний ряд маркерами. Сервер вернет количество столбцов и строчек.

## Получение изображения со стенда
```
http://{stand_api}/image
```

Где 
|аргумент|значение|
|--------|--------|
|`stand_api`|ip адрес стенда в локальной сети|
Получение изображение со стенда. Возвращает актуальную картинку с камеры в формате png.

## Получение обработанных данных со стенда
```
http://{stand_api}/data
```

Где 
|аргумент|значение|
|--------|--------|
|`stand_api`|ip адрес стенда в локальной сети|
Получение обработанных данных об аруко маркерах в формате json

# API получения фейковых данных
Чтобы обеспечить возможность параллельно работать над несколькими программными модулями, был развернут web-сервер, который будет выдавать фейковые данные для визуализации шахматной доски и шахматной логики.

**ВНИМАНИЕ** не нужно постоянно стучаться в api, лучше сохраните полученные в формате json данные для самостоянтельной работы в дальнейшем.

## Как получить фейковые данные для отрисовки шахматной доски
Отправляется GET запрос вида
```
http://{web_server_api}/mock/board/{board_type}
```
Где 
|аргумент|значение|
|--------|--------|
|`web_server_api`|ip адрес web-сервера в локальной сети|
|`board_type`|один из доступных типов доски|

### Типы доски
* `available_moves` - расставлены фигуры и подсвечены ходы для белого коня
* `check` - расставлены фигуры, король черных под шахом
* `desync` - расставлены фигуры. Отсутвует слон на e5
* `stalemate` - патовая ситуация
* `mate` - мат

### Пример данных

|`status`|Список статусов для дополнительных эффектов, все имеют поле `description` для их идентификации|


## Как получить фейковые данные для шахматной логики
Отправляется GET запрос вида
```
http://{web_server_api}/mock/stand/{move_type}
```
Где 
|аргумент|значение|
|--------|--------|
|`web_server_api`|ip адрес web-сервера в локальной сети|
|`move_type`|один из доступных типов доски|

### Типы доски 
* `normal_move` - последовательность ходов пешкой белыми и черными
* `indecisive_player` - последовательность с отменой выбора фигуры 
* `cheater` - неверный ход игроком
* `desync` - последоывтельность с резким изменением колличества фигур на поле
* `horse_check` - последовательность ходов до шаха белым конем
